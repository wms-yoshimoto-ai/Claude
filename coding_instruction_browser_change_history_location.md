# 指示書: 全キャンペーン変更履歴のブラウザ経由取得

## 背景と方針決定

### API検証結果（2026-03-12 確定）

#### Pmax検証

072アカウントのPmax変更履歴（2/10-3/11）をAPIと管理画面CSVで比較した結果:

| カテゴリ | 管理画面CSV | API | 判定 |
|---|---|---|---|
| 地域変更（CAMPAIGN_CRITERION） | 1件（4地域） | 4件 | ✅ 一致 |
| AGステータス変更（ASSET_GROUP） | 5件 | 0件 | ❌ 全欠落 |
| 検索テーマ変更（ASSET_GROUP_SIGNAL） | 6件 | 0件 | ❌ 全欠落 |
| 除外KWリスト適用（CAMPAIGN_ASSET_SET） | 3件 | 0件 | ❌ 全欠落 |
| **合計** | **15件** | **4件** | **73%欠落** |

#### 検索キャンペーン検証（2026-03-12 追加）

072 矯正 all（沖縄）の変更履歴（2025/12/11-2026/03/12）をAPIと管理画面CSVで比較:

| カテゴリ | 管理画面CSV | API | 判定 |
|---|---|---|---|
| 地域変更（CAMPAIGN_CRITERION） | 15件 | 15件 | ✅ 一致 |
| KW変更（AD_GROUP_CRITERION） | 14件 | 0件 | ❌ 全欠落 |
| 広告変更（AD_GROUP_AD） | 10件 | 0件 | ❌ 全欠落 |
| 入札戦略変更（CAMPAIGN） | 5件 | 0件 | ❌ 全欠落 |
| アセット変更（ASSET） | 4件 | 0件 | ❌ 全欠落 |
| 予算変更（CAMPAIGN_BUDGET） | 3件 | 0件 | ❌ 全欠落 |
| 除外KW個別（AD_GROUP_CRITERION） | 3件 | 0件 | ❌ 全欠落 |
| 除外KWリスト適用（CAMPAIGN_ASSET_SET） | 2件 | 0件 | ❌ 全欠落 |
| AG変更・CPN変更・URL変更 | 3件 | 0件 | ❌ 全欠落 |
| **合計** | **59件** | **15件** | **75%欠落** |

※ 2/22の入札戦略変更・除外KWリスト適用は30日以内だが、2/20-3/12期間の再fetchでも返されず。

**結論: `change_event` API はキャンペーンタイプを問わず、CAMPAIGN_CRITERION（地域変更）と AD_GROUP（ステータス変更）しか返さない。**

→ **全キャンペーンの変更履歴はブラウザ経由（管理画面CSV）で取得する。APIでは取得しない。**

※ `fetch_change_history.py` は地域変更・AGステータス変更の補助確認用としてのみ使用可。

## 目的

管理画面の変更履歴レポートから、**全キャンペーン（Pmax＋検索）の変更履歴**をブラウザ経由で取得する。

取得対象（全カテゴリ）:

**Pmax共通:**
1. **地域変更** — 除外地域の追加/削除（具体的な地域名を含む）
2. **AGステータス変更** — アセットグループの有効/一時停止
3. **検索テーマ変更** — アセットグループのヒント（検索テーマ）の追加/削除
4. **除外KWリスト適用** — 共通除外キーワードリストの適用
5. **予算変更** — 日予算の変更
6. **AGアセット変更** — アセットの追加/削除

**検索キャンペーン追加:**
7. **KW変更** — キーワードの追加/削除/入札単価変更/マッチタイプ変更
8. **広告変更** — レスポンシブ検索広告の作成/変更/削除
9. **入札戦略変更** — tCPA・上限CPC等の変更
10. **AG変更** — 広告グループの作成/ステータス変更
11. **除外KW個別** — キャンペーン/広告グループ単位の除外キーワード
12. **CPN変更** — キャンペーン設定の変更
13. **その他** — 上記に該当しないもの

## 取得フロー

### Step 1: 管理画面の変更履歴を開く

URL: `https://ads.google.com/aw/history?ocid={customer_id_without_hyphens}`

例（072）: `https://ads.google.com/aw/history?ocid=1871050145`

### Step 2: フィルタ設定

- **キャンペーン**: 対象キャンペーンを指定（例: 「072 矯正 Pmax（沖縄）」「072 矯正 all（沖縄）」等）
- **期間**: 必要な期間を指定
- **変更元**: 全て（フィルタなし）

### Step 3: 変更履歴の読み取り

管理画面に表示される各レコードから以下の情報を取得:

1. **日時** — 「2026/02/14 6:55:39」形式
2. **ユーザー** — メールアドレス
3. **キャンペーン名**
4. **広告グループ**（あれば）
5. **変更内容** — 全文テキスト（改行含む詳細部分もすべて）

### Step 4: 地域変更の詳細取得

地域変更レコード（「除外地域ターゲティング」「ターゲットを作成」等）は、変更内容の展開部分に具体的な地域名が表示される:

```
4 件の除外地域ターゲティングが追加されました
  与那原町, 沖縄県, 日本（除外）
  八重瀬町, 沖縄県, 日本（除外）
  南城市, 沖縄県, 日本（除外）
  豊見城市, 沖縄県, 日本（除外）
```

**地域名は必ず個別に取得すること**（「4件追加」だけでは不十分）。

### Step 5: 検索テーマの詳細取得

検索テーマ変更レコードは、個々のテーマ名と条件IDが含まれる:

```
3 個のアセット グループのヒントを追加しました
  現在の パラメータ: 「出っ歯 矯正 値段 大学生」
  現在の タイプ: 「検索テーマ」
  現在の条件 ID: 2454208853962
  ...
```

テーマ数が多い場合（40個以上）もすべて取得すること。

## 管理画面CSVの活用

管理画面から直接CSVエクスポートする方法が最も確実で効率的:

1. 変更履歴ページでフィルタ設定（キャンペーン、期間）
2. 右上の「ダウンロード」ボタンでCSVエクスポート
3. CSVの「変更内容」列に全詳細が含まれる

**CSVの構造:**
```
変更履歴レポート
2026年2月10日 - 2026年3月11日
日時,ユーザー,キャンペーン,広告グループ,変更内容
2026/03/11 1:31:02,yoshimoto@wmshome.net,072 矯正 Pmax（沖縄）,,"除外キーワード リスト「cpn共通_7 除外依頼（競合）」を適用しました
  "
```

注意: 変更内容は複数行にまたがる（セル内改行）。CSVパース時に対応が必要。

## カテゴリ分類ルール

| キーワード | カテゴリ |

| `ステータスが` + `アセット グループ` | AGステータス変更 |

## 既存データとの関係

### 既存ブラウザ取得データ

`072_pmax_change_history_20251217_20260209.json` が存在（155レコード）。
ただし以下の課題がある:
- 地域変更の具体的な地域名が取得されていない（「4件追加」のみ）
- `parsed_detail` がない

→ このデータ形式を踏襲しつつ、`parsed_detail` を追加した新フォーマットで出力すること。

### APIデータとの棲み分け

| 対象 | 取得方法 |
|---|---|
| 全キャンペーン変更履歴 | **ブラウザ経由**（本指示書）|
| `fetch_change_history.py` | 地域変更・AGステータス変更の**補助確認用**のみ |

**方針（2026-03-12確定）:** Pmax・検索キャンペーンとも、変更履歴はすべてブラウザ経由で取得する。APIの `change_event` は75%欠落のため本番取得には使用しない。

## 技術的注意事項

1. **CSVのマルチライン**: 変更内容セルは複数行にまたがる。CSVパーサーは改行対応が必要。
2. **ページネーション**: 変更履歴が多い場合、スクロールやページ送りが必要。
3. **複数アカウント対応**: site_id をパラメータ化し、072以外でも使えるようにする。
4. **期間の重複管理**: 既存データ（12/17-2/9）と新規取得（2/10-3/11）の期間重複に注意。
5. **近隣地域ターゲット**: 「沖縄県那覇市安里２丁目６−２７から半径15km」形式もある。

## 補足: 検証で判明した技術的知見

### criterion_id = geo_target_constant ID

- `change_event` の `new_resource.campaignCriterion` は `criterionId` のみ返し、`location.geoTargetConstant` は含まない
- CAMPAIGN_CRITERION の LOCATION 型では **`criterionId` = `geo_target_constant` ID**
- 072での検証: 9194098=与那原町, 9190268=八重瀬町, 9053350=南城市, 9198068=豊見城市 等

### 全キャンペーンタイプで返らないリソースタイプ

API仕様上はサポートリストに含まれるが、実際に返されないもの:

**Pmax固有:**
- `ASSET_GROUP` — AGステータス変更
- `ASSET_GROUP_SIGNAL` — 検索テーマの追加/削除

**全キャンペーンタイプ共通（Pmax＋検索）:**
- `CAMPAIGN` — 入札戦略変更、キャンペーン設定変更
- `CAMPAIGN_BUDGET` — 予算変更
- `CAMPAIGN_ASSET_SET` — 除外KWリスト適用
- `AD_GROUP_CRITERION` — キーワード追加/削除、除外KW個別
- `AD_GROUP_AD` — 広告の作成/変更
- `ASSET` / `CAMPAIGN_ASSET` — アセット変更

**APIが返すもの（2種のみ）:**
- `CAMPAIGN_CRITERION` — 地域変更 ✅
- `AD_GROUP` — 広告グループのステータス変更 ✅

072 矯正 all（検索）実測: 管理画面CSV 59件 → API 15件（25%）。30日以内の入札戦略変更・除外KWリスト適用も返されず。
